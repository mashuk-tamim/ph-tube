Basic JavaScript Concepts

. Discuss the scope of var, let, and const

1. var Scope:

  • Function Scope: Variables declared with var are function-scoped, meaning they are visible throughout the function in which they are declared. 
    function example() {
        if (true) {
            var x = 10; // Function-scoped
        }
        console.log(x); // Accessible here
    }
  • Hoisting: Variables declared with var are hoisted. This means that regardless of where a variable is declared in a function, it is moved to the top of the function during compilation. However, it is initialized with undefined until its actual declaration is reached. 
    function hoistingExample() {
        console.log(a); // undefined
        var a = 5;
        console.log(a); // 5
    }
  • No Block Scope: Unlike let and const, var does not have block-level scope. This means that variables declared with var are not limited to the block in which they are defined. 
    function example() {
        if (true) {
            var x = 10; // Function-scoped
        }
        console.log(x); // Accessible here
    }
  • Global Scope: If var is used outside of any function, it becomes a global variable, visible throughout the entire script. 
    var globalVar = 100; // Global variable
    function example() {
        if (true) {
           console.log(globalVar);
           // Accessible here
        }
    }

2. let Scope:

  • Hoisting: Like var, let variables are hoisted but not initialized. Accessing a let variable before its declaration will result in a ReferenceError. 
    function example() {
         console.log(b);
         // ReferenceError: b is not defined
         let b = 5;
        console.log(b); // 5
    }
  • Block Scope: Variables declared with let have block-level scope. They are confined to the block in which they are declared (inside curly braces {}). 
    function example() {
        if (true) {
            let y = 20; // Block-scoped
        }
        console.log(y);
        // ReferenceError: b is not defined
    }

3. const Scope:

  • Block scope: Variables declared with const also have block-level scope, just like let. They are limited to the block in which they are defined. 
    function example() {
        if (true) {
            const z = 30; // Block-scoped
        }
        console.log(z);
        // ReferenceError: z is not defined
    }
  • Constant Value: const is used to declare constants. Once a value is assigned to a const variable, it cannot be reassigned. 
    const pi = 3.14159; // Constant value
    pi = 3.14; 
    // Error: Assignment to constant variable
  • Immutable vs. Mutable: For objects and arrays declared with const, the variable itself cannot be reassigned, but the properties or elements of the object or array can be modified. 
    const person = { name: 'Alice' };
    person.name = 'Bob';
     // Allowed, updates the property
    person = { name: 'Charlie' };
     // Error: Assignment to constant variable
In practice, it is recommended to use let and const over var because they provide more predictable and safer scoping behavior. let is suitable for variables whose values may change over time, while const is preferred for values that should remain constant. Additionally, the use of const helps convey the intention of a variable being a constant to other developers reading your code.

. Tell us the use cases of null and undefined

1. undefined:

  • Default Value: In JavaScript, variables that have been declared but have not been assigned a value are automatically initialized with undefined. This is often used as a default value when a value is expected but not provided. 
    let x; // x is undefined
  • Function Return: Functions that do not explicitly return a value return undefined by default. 
    function hoistingExample() {
        // no return statement
    }
    const result = example(); // result is undefined
  • Parameter Default Value: When a function parameter is not provided, it is undefined. You can use default parameter values to handle this case. 
    function greet(name = 'Guest') {
        console.log(`Hello, ${name}!`);
    }
    greet(); // Hello, Guest!

2. null:

  • Explicit Absence of Value: null is often used to represent the intentional absence of any object value or value unknown. 
    let user = null;
    // Represents the absence of a user object
  • Reset Value: It can be used to explicitly clear the value of a variable or property. 
    let myVar = 'some value';
    myVar = null; // Clear the value
  • Comparison: null is often used when you want to check if a variable or property has been explicitly set to no value. 
    let data = fetchData();
     // fetchData() returns null if data is not available
    if (data === null) {
        // Handle the absence of data
    }

    • Use Cases Summary:

    • undefined is typically used for variables that have been declared but not assigned, or as the default return value of functions.
    • null is used to explicitly indicate the absence of a value or to reset a value to nothing.
    • Both null and undefined can be used to handle situations where a value is missing or not available.
    • It's important to distinguish between the two. undefined generally represents a lack of initialization, while null represents a deliberate absence of a value.

. What do you mean by REST API?

REST API stands for "Representational State Transfer Application Programming Interface." It is a set of architectural constraints and principles for designing networked applications. REST is not a specific technology or protocol but rather an architectural style that uses HTTP as its underlying communication protocol. RESTful APIs are designed to be simple, scalable, and stateless, making them well-suited for building web services and distributed systems. Here are some key characteristics and concepts of RESTful APIs:

  1. 1. Statelessness: Each HTTP request from a client to a server must contain all the information needed to understand and process the request. The server should not rely on any previous requests from the same client.
  2. 2. Resources: REST, resources are the core abstractions. Resources can represent data entities, such as users, products, or articles, and are typically identified by URLs. Each resource should have a unique URL (Uniform Resource Locator).
  3. 3. HTTP Verbs: RESTful APIs use HTTP verbs (GET, POST, PUT, DELETE, etc.) to perform actions on resources. These verbs indicate the type of operation to be performed on a resource.
    • GET: Retrieve data from the server.
    • POST: Create a new resource on the server.
    • PUT: Update an existing resource or create it if it doesn't exist.
    • DELETE: Remove a resource from the server.
  4. 4. CRUD Operations: RESTful APIs often map to CRUD (Create, Read, Update, Delete) operations, where each HTTP verb corresponds to a specific CRUD operation.
  5. 5. Uniform Interface: RESTful APIs provide a uniform and consistent way to interact with resources through well-defined and predictable URLs and HTTP methods.
  6. 6. Client-Server Architecture:REST separates the client (user interface) from the server (data storage and processing) to improve scalability and portability.
  7. 7. Representation:Resources can have multiple representations, such as JSON, XML, HTML, or others. Clients request a specific representation using the Accept header, and servers respond with the requested representation.
  8. 8. Stateless Communication: RESTful APIs do not maintain any client state on the server between requests. Each request from a client to the server must contain all the necessary information.
  9. 9. Layered System: REST allows for the use of intermediaries, such as load balancers or caching servers, to improve scalability, performance, and security.
  10. 10. HATEOAS (Hypermedia as the Engine of Application State): This principle suggests that a client should be able to navigate a web API dynamically by following links embedded in resource representations, rather than having prior knowledge of all possible endpoints.

RESTful APIs are widely used for building web services, as they are simple to understand, can be easily tested using tools like cURL or Postman, and are compatible with a wide range of programming languages and platforms. They are commonly used for creating web applications, mobile app backends, and connecting various services on the internet.
